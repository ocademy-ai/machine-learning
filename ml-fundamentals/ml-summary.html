15. Summary of machine learning fundamentals#

15.1. Machine Learning Landscape : Discriminative Models#

Most of supervised machine learning can be looked at using the following framework: You have a set of training points \((x_i, y_i)\), and you want to find a function f that ‚Äúfits the data well‚Äù, that is, \(yi \approx f(x_i)\) for most \(i\).

You will start by doing the following:

  • Define the form of \(f\). For instance, we can define \(f = wx + b\), for some constants \(w\) and \(b\). Note that this is a set of functions ‚Äî for different values of \(w\) and \(b\), you will get different functions \(f\), and you want to find an \(f\) from this set that does the ‚Äúbest‚Äù.

  • As you might have noticed, we have been talking about this notion of ‚Äúbest‚Äù, which is ill-defined up to this point. So, we need to make this more concrete. The goal here, as stated above, is to have \(y_i \approx f(x_i)\) for most \(i\).

The above two steps essentially define the function class and the loss function respectively.

Depending on how you choose your function class and the loss function, you get different supervised learning models or even unsupervised learning models:

  • Linear function class with squared-error loss function ‚Äî Linear regression

  • Linear function class with logistic loss function ‚Äî Logistic regression

  • Linear function class with hinge loss ‚Äî SVM

  • Function class containing a network of neurons with cross-entropy loss ‚Äî Neural networks

and so on.
