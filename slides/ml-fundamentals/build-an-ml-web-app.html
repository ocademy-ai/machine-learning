39.15. Build an machine learning web application#

39.15.1. Introduction#

  • Imagine that you‚Äôre working on a machine learning project and you‚Äôve found your champion model. What happens next?

  • Let clients run your Jupyter notebook? Not a good idea!

  • You can convert their notebooks to scripts for somewhat production-grade code

  • And that‚Äôs where Streamlit comes into play

39.15.2. Streamlit#

  • A Python-based library that allows the creation and deployment of machine learning web applications

  • Fast and flexible, turning application development time from days into hours

39.15.3. Display and style data#

39.15.3.1. Use magic#

%pip install streamlit pandas numpy

# My first app
Here's our first attempt at using data to create a table:
"""

import pandas as pd
import numpy as np

df = pd.DataFrame({
  'first column': [1, 2, 3, 4],
  'second column': [10, 20, 30, 40],
})

df

39.15.4. Display and style data#

39.15.4.1. st.write()#

  • st.write() is Streamlit‚Äôs ‚ÄúSwiss Army knife‚Äù

  • You can pass almost anything to st.write(): text, data, Matplotlib figures, Altair charts, and more

  • Streamlit will figure it out and render things the right way

st.write("Here's our first attempt at using data to create a table:")
st.write(pd.DataFrame({
    'first column': [1, 2, 3, 4],
    'second column': [10, 20, 30, 40],
}))

39.15.8. Widgets#

  • When you‚Äôve got the data or model into the state that you want to explore, you can add in widgets like st.slider(), st.button() or st.selectbox()

39.15.9. st.slider()#

x = st.slider('x')  # üëà this is a widget
st.write(x, 'squared is', x * x)

  • On first run, the app above should output the text ‚Äú0 squared is 0‚Äù

  • Then every time a user interacts with a widget, Streamlit simply reruns your script from top to bottom, assigning the current state of the widget to your variable in the process

# You can access the value at any point with:

39.15.13. Layout#

  • Streamlit makes it easy to organize your widgets in a left panel sidebar with st.sidebar. Each element that‚Äôs passed to st.sidebar is pinned to the left, allowing users to focus on the content in your app while still having access to UI controls.

39.15.14. st.sidebar#

# Add a selectbox to the sidebar:
add_selectbox = st.sidebar.selectbox(
    'How would you like to be contacted?',
    ('Email', 'Home phone', 'Mobile phone')
)

# Add a slider to the sidebar:
add_slider = st.sidebar.slider(
    'Select a range of values',
    0.0, 100.0, (25.0, 75.0)
)

39.15.15. st.columns#

left_column, right_column = st.columns(2)

    chosen = st.radio(

        ("Gryffindor", "Ravenclaw", "Hufflepuff", "Slytherin"))
    st.write(f"You are in {chosen} house!")

39.15.16. Show progress#

import time

'Starting a long computation...'

bar = st.progress(0)

for i in range(100):
  # Update the progress bar with each iteration.
  latest_iteration.text(f'Iteration {i+1}')
  bar.progress(i + 1)
  time.sleep(0.1)

'...and now we\'re done!'

39.15.17. Themes#

  • Streamlit supports Light and Dark themes out of the box

  • The ‚ÄúSettings‚Äù menu has a theme editor accessible by clicking on ‚ÄúEdit active theme‚Äù

  • When you‚Äôre happy with your work, themes can be saved by setting config options in the [theme] config section. After you‚Äôve defined a theme for your app, it will appear as ‚ÄúCustom Theme‚Äù in the theme selector and will be applied by default instead of the included Light and Dark themes.

39.15.18. Caching#

  • The Streamlit cache allows your app to stay performant even when loading data from the web, manipulating large datasets, or performing expensive computations

  • The basic idea behind caching is to store the results of expensive function calls and return the cached result when the same inputs occur again rather than calling the function on subsequent runs

  • To cache a function in Streamlit, you need to decorate it with one of two decorators (st.cache_data and st.cache_resource)
